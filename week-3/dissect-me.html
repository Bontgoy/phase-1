Version Control, Tracking Changes, and Using GitHub to Store Your Code

Covering: What are the benefits of version control? How does git help you keep track of changes? Why use GitHub to store your code?

December 26, 2015

Let's talk about version control.

If you've ever used Google Drive, you know what version control is. It is a periodic save of changes you make to a particular document or file. This allows you to later go back and review changes from different sessions and revert to a previous version of the file if necessary. There are a lot of benefits to using version control:

  • You don't have to worry about making mistakes. If you do something wrong, you can always revert to a previous version of the file.
  • Multiple developers can work on a file at the same time. Any conflicts in the code can be resolved before merging the two.
  • You can make notes of each change you make so that later it is clear what changes happened when and for what reason.
  • You can create multiple versions of code and experiment with new code without interfering with the main master branch.

How does git come into all this?

Git is what helps you manage your version control for a project and helps you keep track of changes. Git allows you to do a lot of things:

  • Clone a body of code that other people have already been working on to your local computer
  • Create a new branch of code that is separate from the master branch and does not affect the master branch until it is merged
  • Check the status of changes (added to staging, committed, etc) using the **git status** command.
  • Commit changes to the master branch at periodic intervals as neccessary
  • Add messages to committed changes so that other developers can understand what you did and why.
  • Track changes to the code with the **git log** command
  • Push committed changes to GitHub for other collaborators to review and merge with the master branch.
  • Pull any new changes by collaborators from the master branch on GitHub

Why is GitHub important?

GitHub is an extremely useful way to store your code outside your local computer. GitHub is an online hosting service for all your git code. There are lots of reasons to use it:

  • It makes collaboration easy. Collaborators can create clones of your public code, pull changes to their own computers, or submit a pull request to suggest changes to what you've done.
  • It keeps a copy of your work online. Even if something happens to your computer, you will still have that code
  • You can contribute to open sourced coding projects.
